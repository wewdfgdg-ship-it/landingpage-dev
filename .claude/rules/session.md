# Session Management (세션 관리)

> 이 파일을 레포의 `.claude/rules/session.md`에 복사하세요.

## 세션 이름 규칙

세션 시작 시 작업 내용을 반영한 세션명을 부여한다:
- 형식: `세션 N: [작업명]`
- 예시: `세션 3: T8 레이아웃 구현`, `세션 7: 반응형 버그 수정`

## 세션 번호 관리

1. `.claude/sessions.md` 파일이 **세션 번호의 단일 진실 소스**
2. 새 세션 시작 전 반드시 `sessions.md`를 읽어서 현재 번호 확인
3. 번호는 순차 증가 (건너뛰기 금지)
4. 세션 완료 시 해당 번호를 ✅로 업데이트

## sessions.md 형식

```
📋 [프로젝트명] 세션 목록

✅ 세션 1: [작업명] [순차]
결과: [1줄 요약]

✅ 세션 2: [작업명] [순차, 세션 1 후]
결과: [1줄 요약]

🔄 세션 3: [작업명] [진행중]
프롬프트: [복사용 전체 프롬프트]

⬜ 세션 4: [작업명] [병렬 4-5]
프롬프트: [복사용 전체 프롬프트]

다음 작업: 세션 3 프롬프트를 복사해서 실행하세요.
```

### 상태 이모지
- ⬜ 대기
- 🔄 진행중
- ✅ 완료
- 🚫 블로커로 중단

## 세션 완료 시 필수 실행

### 1. sessions.md 업데이트
- 현재 세션: ⬜ → ✅ + 결과 요약 추가
- 다음 세션: "다음 작업" 안내 업데이트

### 2. 커밋 메시지 형식
```
✅ [프로젝트] 세션 N 완료
결과: [1줄 요약]
다음: 세션 N+1 프롬프트를 복사해서 실행하세요.
```

### 3. Push + 알림
- 커밋 후 반드시 push
- 텔레그램 환경변수 있으면 직접 알림, 없으면 GitHub Actions가 처리

## 세션 쪼개기 규칙

프로젝트 분석 후 세션을 쪼갤 때:
1. 1세션 = 컨텍스트 50% 이내로 끝나는 작업량
2. 의존성 표시 필수 (순차/병렬 구분)
3. 각 세션에 **복사+붙여넣기용 프롬프트** 포함
4. 결과를 sessions.md에 저장 후 push
