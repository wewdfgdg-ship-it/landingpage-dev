# 반복 메커니즘

## 루프 발동 조건

### 자동 발동
- reviewer.md에서 **FAIL** 판정
- `npx tsc --noEmit` 에러 > 0
- `npm run lint` 에러 > 0
- `npm run build` 실패
- 품질 게이트 미달 (카피 < 80점)

### 수동 발동
- 사용자 `--loop` 플래그 지정
- 사용자 "다시", "개선", "수정" 등 반복 요청

## 루프 프로세스

```
FAIL 판정 또는 에러 발생
    │
    ▼
[이터레이션 N/3]

1️⃣ 문제 식별
   ├── 에러 메시지 수집 (tsc, lint, build 출력)
   ├── 검수 피드백 수집 (reviewer.md 결과)
   └── 실패 항목 목록 작성
    │
    ▼
2️⃣ 원인 분석
   ├── 단순 에러 → 에러 메시지에서 직접 파악
   ├── 복잡 에러 → Sequential MCP (--seq) 활용
   └── 패턴 에러 → memory.md 이전 실수 참조
    │
    ▼
3️⃣ 수정 실행
   ├── Edit/Write로 코드 수정
   ├── 변경 내용 기록
   └── 수정 근거 기록
    │
    ▼
4️⃣ 검증
   ├── checklist.md 재실행
   │   ├── npx tsc --noEmit
   │   ├── npm run lint
   │   └── npm run build
   └── 전부 통과?
       ├── YES → 5단계로
       └── NO → 남은 에러로 다시 1단계
    │
    ▼
5️⃣ 재검수
   └── reviewer.md 판정
       ├── PASS ✅ → 루프 탈출
       ├── WARN ⚠️ → 루프 탈출 (개선 사항 기록)
       └── FAIL ❌ → 다음 이터레이션
```

## 탈출 조건

### 성공 탈출
- reviewer.md PASS 또는 WARN 판정
- checklist.md 전 항목 통과

### 강제 탈출
- **최대 반복 3회** 도달
- 에스컬레이션: 사용자에게 상황 보고 + 수동 판단 요청

### 에스컬레이션 보고 포맷
```
🚨 루프 3회 반복 후에도 미해결

문제: [에러 내용]
시도한 수정:
  - 이터레이션 1: [수정 내용] → [결과]
  - 이터레이션 2: [수정 내용] → [결과]
  - 이터레이션 3: [수정 내용] → [결과]

남은 에러: [현재 상태]
제안: [가능한 해결 방향]
```

## 특수 루프: 카피 품질 게이트

엔진 ⑤ 내부에 자체 루프 존재:
```
카피 생성
    │
    ▼
품질 게이트 (frame 60% + tone 40%)
    │
    ├── ≥ 80점 → 통과
    └── < 80점
        ├── 실패 섹션만 추출
        ├── buildRetryPrompt() → 피드백 프롬프트
        ├── AI 재생성 (실패 섹션만)
        ├── mergeCopy() → 병합
        └── 재평가 (max 2회)
            ├── 통과 → 진행
            └── 2회 실패 → 현재 결과로 진행 + ⚠️ 경고
```

## 학습 누적

매 이터레이션 종료 시 memory.md에 기록:

```markdown
### 루프 기록 [날짜]
- **문제**: [에러 설명]
- **원인**: [근본 원인]
- **해결**: [수정 내용]
- **교훈**: [같은 실수 방지를 위한 학습]
```

## 메트릭

| 메트릭 | 정상 | 주의 | 위험 |
|--------|------|------|------|
| 이터레이션 수 | 1 | 2 | 3 (에스컬레이션) |
| 에러 감소율 | 매 이터레이션 50%+ 감소 | 30% 미만 | 증가 (접근법 변경 필요) |
| 동일 에러 재발 | 없음 | 1회 | 2회+ (memory.md 참조 실패) |
