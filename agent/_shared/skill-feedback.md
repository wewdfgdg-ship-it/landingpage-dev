## 기록 규칙

### 에이전트 기록 의무

```
스킬 사용 완료
    │
    ├── 성공 → 이 파일에 ✅ + 메모 1줄
    ├── 실패 → 이 파일에 ❌ + 실패 사유 1줄
    └── 체인 → 체인ID(CHAIN-NNN) + 전체 결과
```

### 필수 기록 항목

| 필드 | 내용 | 예시 |
|------|------|------|
| 날짜 | YYYY-MM-DD | 2026-03-10 |
| 스킬ID | SK-NNN 또는 CHAIN-NNN | SK-001, CHAIN-002 |
| 엔진 | NN-이름 | 10-code, 05-copy |
| 결과 | ✅ 성공 / ❌ 실패 | ✅ 성공 |
| 문제유형 | skill-routing.md 분류 | 코드, 생성, 분석 |
| 메모 | 성공 이유 또는 실패 사유 1줄 | "한국어 미지원" |

## Tool Intelligence 처리 규칙

> Tool Intelligence는 탐색 시 이 파일을 읽고 아래 규칙에 따라 skill-catalog.md를 갱신한다.

| 패턴 | Tool Intelligence 행동 |
|------|----------|
| ✅ 3회+ 연속 | skill-catalog.md 성공률 갱신, 적합 엔진(for_agents) 확장 검토 |
| ❌ 3회+ 연속 | skill-catalog.md status → deprecated, 대체 탐색 트리거 발동 |
| ❌ 실패 메모에서 키워드 추출 | scan-targets.md 반영, 해당 키워드로 새 탐색 시작 |
| 새 엔진에서 ✅ | skill-catalog.md for_agents 목록에 해당 엔진 추가 |
| 30일간 사용 0 | "필요한가?" 재검토 — 불필요하면 status 유지, 필요하면 에이전트에 알림 |
| "해결 불가" 기록 | 즉시 Tool Intelligence 탐색 트리거 (조건부 실행 ⑤번) |

### 키워드 추출 규칙

```
실패 메모 분석
    │
    ├── "한국어 미지원" → 키워드: "korean support"
    ├── "출력 포맷 불일치" → 키워드: "structured output"
    ├── "속도 느림" → 키워드: "fast [문제유형]"
    ├── "정확도 낮음" → 키워드: "accurate [문제유형]"
    └── 기타 → 실패 사유에서 핵심 명사 추출
```
