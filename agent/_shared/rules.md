# 공통 운영 규칙 — 전 에이전트 적용

> 모든 에이전트(00~12)가 준수해야 하는 공통 규칙.
> 에이전트별 추가 규칙은 각자 `rules.md`에 정의한다.

---

## 1. 버전 체크 절차

### 필수 시점
- 새 패키지 추천/도입 전
- 기존 패키지 업데이트 검토 시
- Tool Intelligence 3단계 품질 평가 통과 후 (브로드캐스트 전)

### 절차
```
1. npm info {패키지명} version          → 최신 버전 확인
2. npm info {패키지명} engines          → Node.js 호환성
3. npm info {패키지명} peerDependencies → 의존성 충돌 확인
4. npm info {패키지명} time             → 마지막 배포일 확인
```

### 호환성 기준
| 항목 | 필수 조건 |
|------|----------|
| Node.js | >=18 (프로젝트 기준) |
| Next.js | 16.x 호환 |
| TypeScript | strict 모드 호환 |
| 마지막 배포 | 6개월 이내 |

### 실패 시
```
→ 버전 체크 실패한 패키지는 추천/도입 불가
→ memory.md에 실패 사유 기록
→ 대안 패키지 탐색
```

## 2. 효율 로그 강제 Write 규칙

### 원칙
> **모든 에이전트는 작업 완료 시 효율 로그를 memory.md에 반드시 Write한다.**
> 결과가 0건이어도, 실패해도, "탐색 불필요"여도 기록한다.

### 로그 포맷
```
## 세션 히스토리

| 날짜 | 작업 | 결과 | 소요시간 | 학습 |
|------|------|------|---------|------|
| {YYYY-MM-DD} | {작업 요약} | {성공/실패/스킵} | {분} | {한 줄 학습} |
```

### 필수 기록 항목
| 항목 | 설명 |
|------|------|
| 날짜 | 작업 수행일 |
| 작업 | 무엇을 했는지 (한 줄) |
| 결과 | 성공/실패/스킵/부분성공 |
| 소요시간 | 대략적 소요 시간 (분 단위) |
| 학습 | 다음에 더 잘할 수 있는 포인트 (없으면 "-") |

### 위반 시
```
→ 로그 누락 = 작업 미완료로 간주
→ 3회 연속 누락 시 해당 에이전트 규칙 강화 플래그
```

## 3. 소스 성공률 자동 하락 규칙

### 적용 대상
- source-registry.md의 모든 소스 (Tier 1~3)

### 자동 하락 조건
| 조건 | 행동 |
|------|------|
| 성공률 50% 미만 (5회 이상 탐색) | 자동 Tier 하락 (1→2, 2→3) |
| 성공률 30% 미만 (5회 이상 탐색) | 즉시 Hold 이동 |
| 3회 연속 결과 0건 | 즉시 Hold 이동 |
| Hold 90일 경과 | 재평가 (Tier 3에서 재시작 또는 삭제) |

### 자동 승격 조건
| 조건 | 행동 |
|------|------|
| 성공률 80% 이상 (5회 이상 탐색) | 자동 Tier 승격 (3→2, 2→1) |
| 10회 연속 유효 결과 | 즉시 Tier 승격 |

### 계산 방법
```
성공률 = (유효 결과 건수 / 총 탐색 횟수) × 100

유효 결과 = 4단계 필터 2단계(적합성) 이상 통과한 후보 수
총 탐색 횟수 = 해당 소스에서 검색을 실행한 총 횟수
```

### 기록
```
→ Tier 변경 시 source-registry.md 즉시 갱신
→ memory.md "소스 변경 이력"에 변경 사유 기록
```

## 4. 파일 수정 권한

### 공통 수정 가능
| 경로 | 수정 가능 에이전트 |
|------|-----------------|
| `agent/{자기번호}/` | 해당 에이전트만 |
| `agent/_shared/` | 13-tool-intelligence만 (브로드캐스트) |

### 공통 수정 금지
| 경로 | 사유 |
|------|------|
| `src/` | 코드 수정은 10-code-engine만 |
| `.env`, `.env.local` | 환경변수 직접 수정 금지 |
| `package.json` | 패키지 추가/삭제는 사용자 승인 필수 |
| `next.config.ts` | 설정 변경은 사용자 승인 필수 |

## 5. MCP 비용 의식

> **@mcp-cost-rules.md** 상세 규칙 참조.

### 핵심 요약
```
- MCP 기본 상태 = OFF
- 필요한 작업에서만 ON → 완료 즉시 OFF
- 동시 2개+ ON 금지 (특수 케이스 제외)
- 5호출 초과 ON 유지 시 방법 재검토
- MCP 추천 시 비용 경고문 필수 첨부
```

## 6. 세션 관리

### 세션 시작
```
1. memory.md 읽기 (CHECKPOINT 확인)
2. tool-broadcast.md 확인 (신규 추천 여부)
3. 조건 판별 (에이전트별 Step 0)
4. 이전 세션 학습 사항 확인
5. 작업 실행
```

### 세션 종료
```
1. 효율 로그 Write (필수 — 생략 불가)
2. memory.md CHECKPOINT 갱신
3. 미완료 작업 있으면 블로커 기록
```

## 업데이트 규칙

- 새 공통 규칙 발견 시 → 이 파일에 추가
- 에이전트 전용 규칙 → 각자 rules.md에 추가 (여기 아님)
- 3개+ 에이전트에 동일 규칙 적용 시 → 여기로 통합
- 충돌 시 → 이 파일(공통) > 에이전트별 rules.md
