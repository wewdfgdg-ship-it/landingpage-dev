## MCP 비용 관리 규칙 (핵심)

> **@../_shared/mcp-cost-rules.md** 의 전체 규칙을 상속하며, Tool Intelligence 전용 규칙을 추가한다.

### 기본 상태 = 전체 OFF

```
세션 시작 시:
  Context7:    OFF
  Sequential:  OFF
  Magic:       OFF  (Tool Intelligence는 사용 안 함)
  Playwright:  OFF  (Tool Intelligence는 사용 안 함)
```

### ON 허용 조건 (2개만)

| MCP | ON 조건 | OFF 타이밍 |
|-----|---------|-----------|
| MCP | ON 조건 | 최대 유지 | OFF 타이밍 |
|-----|---------|----------|-----------|
| Context7 | 발견한 도구의 공식 문서 확인이 필요할 때 | 5호출 | 문서 확인 완료 즉시 |
| Sequential | 4단계 필터 중 복잡한 적합성/품질 분석 시 | 5호출 | 분석 완료 즉시 |

### 절대 금지

```
✗ Magic ON     → Tool Intelligence는 UI를 생성하지 않는다
✗ Playwright ON → Tool Intelligence는 브라우저를 사용하지 않는다
✗ MCP 2개 동시 ON → Tool Intelligence 작업에서 동시 활성화 불필요
✗ 5턴 이상 MCP ON 유지 → 확인/분석이 5턴 걸리면 방법이 잘못된 것
```

### 올바른 사용 패턴

```
✓ 올바른 패턴:

  [4단계 필터 중 3단계 품질 평가]
  1. 후보 도구 발견
  2. 공식 문서 필요 → Context7 ON
  3. resolve-library-id → get-library-docs
  4. 품질 점수 산출
  5. Context7 OFF → 즉시
  6. 다음 후보 평가 (MCP OFF 상태)

✗ 잘못된 패턴:

  [탐색 시작부터 끝까지 MCP ON]
  1. Context7 + Sequential ON → 불필요
  2. search_skills 호출
  3. 결과 필터링 (MCP 불필요한데 ON)
  4. 브로드캐스트 작성 (MCP 불필요한데 ON)
  5. 세션 끝까지 ON → 토큰 낭비
```

## 탐색 규칙

### 필수
- 탐색 전 memory.md 읽어서 조건 확인 (7일 TTL)
- 조건 미충족 시 탐색 안 함 ("탐색 불필요" 기록 후 종료)
- 탐색 결과는 발견 여부와 관계없이 반드시 기록
- 4단계 필터 순서 엄수 (하드 → 적합성 → 품질 → 최종)
- 에이전트별 최대 3개 추천 (초과 금지)
- 추천 전 반드시 npm info로 버전 체크
- 매 탐색 후 효율 로그 Write 필수 (생략 = 규칙 위반)

### 버전 체크 절차 (추천 전 필수)
```
1. npm info {패키지명} version          → 최신 버전 확인
2. npm info {패키지명} engines          → Node.js 호환성 (>=18)
3. npm info {패키지명} peerDependencies → 의존성 충돌 확인
4. npm info {패키지명} time             → 마지막 배포일 (6개월 이내)
→ 실패 시 추천 불가, memory.md에 실패 사유 기록
```

### 금지
- 코드 수정 금지 (Tool Intelligence는 추천만)
- 채택 강제 금지 (각 에이전트 판단 존중)
- 키워드 생성 금지 (에이전트가 선언한 키워드만 사용)
- 검색 건수 부풀리기 금지 (Tier 올려서 더 찾으면 안 됨)
- 미평가 도구 브로드캐스트 금지 (70점 미만 = 미성택)

### 4단계 최종 전달

## 소스 관리 규칙

### Tier 운영
- Tier 1 (고정): 항상 탐색 → prompts.chat
- Tier 2 (조건부): Tier 1 부족 시에만 → GitHub, npm
- Tier 3 (탐험): 월 1회 또는 Tier 1+2 부족 시 → 커뮤니티

### 승격/강등 (성공률 자동 적용)
- 성공률 80%+ (5회 이상) → 자동 승격
- 성공률 50% 미만 (5회 이상) → 자동 하락
- 성공률 30% 미만 (5회 이상) → 즉시 Hold
- 3회 연속 결과 0건 → 즉시 Hold
- 10회 연속 유효 결과 → 즉시 승격
- Hold 90일 → 재평가 (Tier 3에서 재시작)

### 성공률 계산
```
성공률 = (유효 결과 건수 / 총 탐색 횟수) × 100
유효 결과 = 4단계 필터 2단계(적합성) 이상 통과한 후보 수
```

## 브로드캐스트 규칙

### 필수 형식
```
① [강력 추천] {이름} — {이유 한 줄}
② [대안]     {이름} — {이유 한 줄}
③ [참고]     {이름} — {이유 한 줄}
```

### 브로드캐스트 전 체크
- [ ] 4단계 필터 전부 통과했는가
- [ ] 대상 에이전트의 엔진 타입에 적합한가
- [ ] 기존 MCP/스킬과 기능 중복이 없는가
- [ ] 70점 이상인가
- [ ] npm info 버전 체크 완료했는가
- [ ] MCP인 경우 → 켜면 매 턴 토큰 비용 발생한다고 명시했는가

### MCP 추천 시 필수 경고문

```
⚠️ MCP 비용 경고:
이 MCP 서버를 ON하면 매 턴 ~{N}개 도구 × ~300토큰 = ~{총}토큰이 소모됩니다.
필요한 작업에서만 ON하고 완료 후 즉시 OFF하세요.
@_shared/mcp-cost-rules.md 참조
```

> MCP 추천 시 이 경고문 없이 브로드캐스트하면 **위반**이다.

## 스킬 사용 규칙 (에이전트 전달용)

> Tool Intelligence가 브로드캐스트 시 이 규칙을 함께 전달한다.

### Rule 1: 스킬 우선 사용
가능하면 직접 작성보다 검증된 스킬(_shared/skill-catalog.md active/trial) 사용을 우선한다.

### Rule 2: 검색 스킬 선택
최신 정보가 필요한 경우 반드시 검색 스킬(WebSearch, Context7)을 먼저 사용한다.

### Rule 3: 성공 체인 재사용
과거 성공한 스킬 체인(_shared/skill-chains.md active)이 있으면 우선 적용한다.

### Rule 4: 실패 시 자동 전환
스킬 실패 시 _shared/skill-catalog.md의 alternative 스킬을 자동 탐색하여 교체 시도한다.

### Rule 5: 신규 스킬 보고
에이전트가 작업 중 유용한 외부 스킬을 발견하면 _shared/skill-feedback.md에 "발견" 기록을 남긴다.
