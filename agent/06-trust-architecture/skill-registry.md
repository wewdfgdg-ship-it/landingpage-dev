# 스킬 레지스트리 — Trust Architecture Agent

## 이 에이전트의 스킬 우선순위

| 순위 | 스킬 | 주요 용도 | 사용 빈도 |
|------|------|----------|----------|
| 1 | **/implement** | 엔진 코드 구현 (index.ts, types.ts, rules.ts) | 높음 |
| 2 | **/test** | 규칙 매칭 단위 테스트, 범위 검증 | 중간 |
| 3 | **/analyze** | 규칙 로직 분석, 타입 호환 검증 | 낮음 |
| 4 | **/improve** | 규칙 구조 개선, 리팩토링 | 낮음 |
| 5 | **/troubleshoot** | 규칙 매칭 오류 디버깅 | 낮음 |

## 내장 스킬 매핑

### /implement — 기능 구현
- **자동 호출**: 새 엔진 구현 요청
- **이 엔진**: rules.ts (규칙 정의) + index.ts (실행 로직) + types.ts (타입)

### /test — 테스트
- **자동 호출**: 규칙 매칭 테스트, 범위 검증 테스트

### /analyze — 코드 분석
- **자동 호출**: 규칙 로직 검증

### /build — 프로젝트 빌드
- **자동 호출**: 코드 변경 완료 후 검증

## 자동 호출 판단 로직

```
사용자 요청 분석
    │
    ├── "구현/만들어/추가" → /implement
    ├── "테스트/검증" → /test
    ├── "분석/점검/리뷰" → /analyze
    ├── "개선/최적화" → /improve
    ├── "에러/버그/안됨" → /troubleshoot
    └── "빌드/배포" → /build
```

## 스킬 조합 패턴

### Trust Architecture 엔진 구현
```
/implement (rules.ts + index.ts + types.ts)
    → /test (규칙 매칭 테스트)
    → /build (빌드 검증)
```
